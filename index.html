  1. Set 数据结构

    ES6 提供了新的数据结构Set。它类似于数组，但是成员的值都是唯一的，没有重复的值。

    Set本身是一个构造函数(并不是数组)，用来生成Set数据结构。

    Set 结构的实例有以下属性：

    • Set.prototype.constructor: 构造函数，默认就是Set函数。
    • Set.prototype.size: 返回Set实例的成员总数

    Set 实例的方法(部分)：

    • add(value): 添加某个值，返回Set结构本身。

    • delete(value): 删除某个值，返回一个布尔值，表示删除是否成功。

    • has(value): 返回一个布尔值，表示该值是否为Set的成员。

    • clear(): 清除所有成员，没有返回值。

    • keys(): 返回键名的遍历器

    • values(): 返回键值的遍历器

    • entries(): 返回键值对的遍历器

      Set的遍历顺序就是插入顺序，比如，使用Set保存一个回调函数表，调用时就能保证按照添加顺序调用

      keys方法、values方法、entries方法返回的都是遍历器对象，由于Set结构没有键名，只有键值，所以Keys方法和values方法的行为完全一致, entries方法返回的遍历器，同时包括键名和键值,所以每次输出一个数组，它的两个成员完全相等。

    • forEach(): 使用回调函数遍历每个成员

Can Place Flowers

Veröffentlicht am 2018-12-19 | in leetcode study everyday

Suppose you have a long flowerbed in which some of the plots are planted and some are not. However, flowers cannot be planted in adjacent plots - they would compete for water and both would die.

Given a flowerbed(represented as an array containing 0 and 1,where 0 means empty and 1 means not empty), and a number n, return if n new flowers can be planted in it without violating the no-adjacent-flowers rule.

Example 1:

Input: flowerbed = [1,0,0,0,1], n = 1
Output: True

Example 2:

Input: flowerbed = [1,0,0,0,1], n = 2
Output: False

Note:
1.The input array won’t violate no-adjacent-flowers rule.
2.The input array size is in the range of [1,20000].
3.n is a non-negative integer which won’t exceed the input array size.

Solution: 

var canPlaceFlowers = function(flowerbed,n){
  if(n < 1){
    return false
  }
  if(n > parseInt(flowerbed.length/2 + 1)){
    return false
  }
  for(let i = 0;i < flowerbed.length;i++){
    if(flowerbed[i] === 0 && !flowerbed[i-1] && !flowerbed[i+1]){
      flowerbed[i] = 1;
      n--
    }
  }
  return n<=0
}

Veröffentlicht am 2018-12-19 | in regular expression

正则表达式是匹配模式，要么匹配字符，要么匹配位置。

一、匹配字符

1.两种模糊匹配：横向模糊匹配、纵向模糊匹配

横向匹配：/ab{2,5}c/

纵向匹配： /a[123]b/

2.字符组

范围表示法: [1-6a-fG-M]

排除字符组: [^abc]

常见缩写: \d表示[0-9], \D表示[^0-9], \w表示[0-9a-zA-Z], \W表示[^0-9a-zA-Z], \s表示[ \t\v\r\f],表示空白符，包括空格、水平制表符、垂直制表符、换行符、回车符、换页符，\S表示[^ \t\v\n\r\f], .是通配符，表示除换行符、回车符、行分隔符和段分隔符外的任意字符

匹配所有字符: [\d\D], [\w\W], [\s\S], [^]

3.量词

{m,}表示至少出现m次

{m}等价于{m,m},表示出现m次

？等价于{0,1}

+ 等价于{1,}

* 等价于{0,}

惰性匹配：再量词后面加上问号，如{m,n}?, +?

4.多选分支

具体形式:(p1|p2|p3)

多选分支结构是惰性的，即当前面的匹配上了，后面就不再尝试，例如 “goodbye”.match(/good|goodbye/),匹配结果为[“good”]; “goodbye”.match(/goodbye|good/),匹配结果为[“goodbye”]

二、匹配位置

1.匹配位置的锚字符

^ $ \b \B (?=p) (?!p) (?<=exp) (?<!exp) (?<exp) (?<!exp)

^ 匹配开头，在多行匹配中匹配行开头

$ 匹配结尾，在多行匹配中匹配行结尾

\b 单词边界，包括\w和\W之间的位置，\w和^之间的位置，\w和$之间的位置

\B 非单词边界, 包括\w与\w、\W与\W、^与\W、\W与$之间的位置

(?=exp) 匹配exp前面的位置,零宽度正预测先行断言

(?!exp) 非exp前面的位置,零宽度负预测先行断言

(?<exp) 匹配exp后面的位置,零宽度正回顾后发断言

(?<!exp) 匹配前面不是exp的位置,零宽度负回顾后发断言

2.位置的理解

位置可以理解先满足该条件，才能继续匹配

例如:/^hello$/写成/^^hello$$$/,或者写成/(?=he)^^he(?=\w)llo$\b\b$/

不匹配任何东西的正则 /.^/

数字的千位分隔符表示法: “123456789”.replace(/(?!^)(?=(\d{3})+$)/g,”,”)

验证密码问题: 要求密码长度6-12位，由数字、小写字符和大写字符组成，但必须至少包括两种字符

/((?=.*[0-9])(?=.*[a-z])|(?=.*[0-9])(?=.*[A-Z])|(?=.*[a-z])(?=.*[A-Z]))^[0-9A-Za-z]{6,12}$/

/(?!^[0-9]{6,12}$)(?!^[a-z]{6,12}$)(?!^[A-Z]{6,12}$)^[0-9A-Za-z]{6,12}$/

三、括号的作用

1.分组和分支结构：(ab)+ (a|b)

2.引用分组

1.提取数据

match(), exec(), $1~$9

2.替换

“2017-06-12”.replace(/(\d{4})-(\d{2})-(\d{2})/, “$2/$3/$1”)

3.反向引用

写一个正则支持匹配如下三种格式

2016-06-12 2016/06/12 2016.06.12

/\d{4}(-|\/|\.)\d{2}\1\d{2}/

其中\1表示应用之前的分组(-|\/|\.),不管它匹配到什么，\1都匹配那个同样的字符. \2,\3同理

注：括号嵌套如何决定\1, \2, \3: 以左括号为准

四、回溯

1.没有回溯的匹配

假设正则为/ab{1,3}c/，当目标字符串是”abbbc”时，其匹配过程是: a→ab→abb→abbb→abbbc

2.有回溯的匹配

假设正则为/ab{1,3}bbc/，目标字符串是”abbbc”，匹配过程是:a→ab→abb→abbb→abbbb(不匹配，回溯)→abbbb(不匹配，继续回溯)→abbbc

3.Js中产生回溯的地方

1.贪婪量词

当多个贪婪量词挨着存在，并相互有冲突时，遵循先下手为强，因为深度优先搜索

2.惰性量词

惰性量词就是在贪婪量词后面加个问号。表示尽可能少的匹配

惰性量词同样有回溯现象，比如:

其匹配过程为 1→12→123→1234→1234(不匹配$,回溯)→1→12→123→1234→12345

3.分支结构

分支结构也是惰性的，比如/can|candy/匹配”candy”，得到的结果是“can”，它也有回溯

五、正则表达式拆分

1.结构和操作符

字面量: 匹配一个具体字符，包括不用转义和需要转义的。

字符组: 匹配一个字符，可以是多种可能之一，比如[0-9]

量词: 表示一个字符连续出现，比如a{1,3}

锚点: 匹配一个位置, 比如^、\b

分组: 用括号表示一个整体，比如(ab)+，非捕获分注(?:ab)+

分支: 多个子表达式多选一，比如abc|bcd

反向引用: 比如\1，表示引用第1个分组。

其中涉及到的操作符有:

1.转义符\

2.括号和方括号(…)、(?…)、(?=…)、(?!…)、[…]

3.量词限定符{m}、{m,n}、{m,}、?、*、+

4.位置和序列^、$、\元字符、一般字符

5.管道符(竖杠) |

上面操作符的优先级从上至下，由高到低

例: /ab?(c|de*)+|fg/

2.注意要点

1.匹配字符串整体问题

例: 比如要匹配字符串”abc”或者”bcd”，如果一不小心就会写成/^abc|bcd$/,因为位置字符和字符序列优先级要高于竖杠，其等价于/(?:^abc)|(?:bcd$)/，应修改为/^(abc|bcd)$/

2.量词连缀问题

假设有这样的匹配要求: 每个字符为a、b、c任选其一，字符串的长度是3的倍数

不能写成/^[abc]{3}+$/，要写成/^([abc]{3})+$/

3.元字符转义问题

元字符，即正则中有特殊含义的字符
